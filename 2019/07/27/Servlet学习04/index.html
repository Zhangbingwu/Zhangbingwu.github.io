EL表达式

EL表达式的作用

  • 可以获取servlet四个作用域中的数据
  1. request
  2. session
  3. pageContext
  4. application

注意：不是获取servlet流转的数据，而是四个作用域中的数据。

使用 ${表达式}

如果是请求参数

  1. ${param.键名}
  2. ${paramValues.键名} 返回数组，例如爱好。

如果是setAttribute()存储的数据

  • ${键名}
    存储的式字符串返回字符串
    存储的对象返回对象
  • ${键名.属性名} 直接返回对象的属性值
  • ${list} 如果是List集合,返回数组
    ${list[0]} 集合中第一个对象。
  • ${map.map的键名} 如果是Map集合，返回该map键名对应的值
    ${map.get(键名)}

EL表达式作用域查找顺序

  • 如果请求数据中，不同作用域键名相同。EL表达式查找哪个？
  • 默认顺序：
    作用域从小到大找，小的里面没找到再去大的里面找，找到了后就不再继续查找。
    作用域从小到大：pageContext（当前页面）->request（一次请求）->session（一次会话
    ）->application（服务器启动到关闭）
    注意：从小到大依次找。
  • 指定顺序：
    想要EL表达式去指定的作用域查找怎么办？
    ${pageScope.键名}
    ${requestScope.键名}
    ${sessionScope.键名}
    ${applicationScope.键名}

EL表达式还可以做简单的逻辑运算

  • 加减乘除取余
    ${1+2} — 3
  • 判断
    ${1>2} — true
    ${1==1} — true
    ${1!=2} — true
    ${1<=1} — true
  • 三目运算
    ${sex==1?”男”:”女”}
  • 注意：
  1. EL表达式中’+’号只有运算的作用，没有字符串连接作用,会报错。
  2. ${1+”2”} 会将”2”转化为2，继续运算。

EL获取请求头数据和Cookie数据

请求头数据

  • ${header} 取出所有请求头数据
  • ${header[“键名”]} 取出指定键名的请求头数据
  • ${headerValues[“键名”]} 取出指定键名(同键不同值)的请求头数组
    ${headerValues[“键名”][0]}

Cookie数据

  • ${cookie} 取出所有Cookie数据
  • ${cookie.键名} 获取这个Cookie对象

JSTL标签

使用之前

  • 导jstl的jar包
    下载impl
  • 设置page指令
    <%@taglib prefix=”c” uri=”http://java.sun.com/jsp/jstl/core" %>

jstl基本标签

  • <c:out value="" default="" ></c:out>
    结合EL表达式<c:out value="${str2}" default="天天心情好"></c:out>
    作用：直接输出，如果为空，则输出默认值。

  • <c:set var="键名" value="键值" scope="存储作用域"></c:set>
    类似于 作用域对象.setAttribute(“”,””)，默认存放在pageContext作用域

  • <c:remove var="键名" scope="存储作用域"></c:remove>
    类似于作用域对象.removeAttribute()
    注意：如果不指定作用域，会将四个作用域中符和键名的数据都删除。
