AOP

什么是AOP？

  • AOP (Aspect Oriented Programing)

  • 面向切面编程

  • 理解

  1. 正常的程序执行流程都是纵向的，自上而下。
  2. 在原有的纵向执行流程中添加横切面。
  3. 在程序原有纵向执行流程中,针对某一个或某一些方法添加通知,形成横切面过程就叫做面向切面编程.

特点

  • 1.不需要修改原有的代码。
  • 2.使程序具有高扩展性。
  • 3.原有功能释放了部分逻辑，使职责更加明确。
    (将原来复杂的功能(方法)中部分逻辑代码在其前后进行分担)

常用概念

  • 原有功能: 切点,pointcut
  • 前置通知: 在切点之前执行的功能.beforeadvice
  • 后置通知: 在切点之后执行的功能,afteradvice
  • 如果切点执行过程中报出异常,会触发异常通知.throwsadvice
  • 所有功能总称叫做 切面.
  • 织入: 把切面嵌入到原有功能的过程叫做 织入

spring提供了2种AOP实现方式

Schema-based

  • 每个通知都需要实现接口或类
  • 配置 spring 配置文件时在<aop:config>配置
  1. 通配符*
  2. (..)匹配任意参数
    例：想要拦截com.zbw.test包下的所有类可以：(* com.zbw.test.*.*(..))

AspectJ

  • 每个通知不需要实现接口或类
  • 配置 spring 配置文件是在<aop:config>的子标签<aop:aspect>中配置

注意：

  1. after就算切点报异常也会执行 类似finnaly
  2. after-returning 切点正常执行才会执行
  3. 执行顺序和配置顺序有关系。
  • 带参数的配置
    注意：
  1. 切点配置参数在execution外，用and连接，不能用&&，（xml的特殊符号）
    spring在转化成java后会转换成&&。
  2. 参数的名称必须都相同，包括通知类中的方法参数名称。
